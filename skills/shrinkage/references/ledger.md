# The Shrinkage Ledger — durable keeps / exclusions / frozen files

Institutional memory the **tool** owns, not the session. Without it, every audit
re-receives the same context by hand and every new plan re-copies keeps forward
— fragile: forget one frozen file and you corrupt a sealed subsystem; forget a
keep and settled items get re-flagged forever.

Location (first that exists, resolved by `scripts/ledger.py`):
`.shrinkage/ledger.md`, `SHRINK-LEDGER.md`, or `.planning/SHRINK-LEDGER.md`.

Three sections. In `## frozen` and `## excluded`, each row's **first token** is a
path or glob (directory paths end with `/`; `**` matches recursively); the rest
of the row is a human-readable reason. `## keeps` rows are free-form.

## frozen

Files the tools must **NEVER edit**. `safe_commit.py` hard-refuses any commit
touching a frozen path, and audit excludes them from candidate lists. Use for
hash-sealed subsystems, generated files — anything a mere textual edit breaks.

```
## frozen
src/LAB/**             content-hash sealed — any textual edit invalidates the seal
config/generated.php   regenerated by build; manual edits are lost
```

## excluded

Globs excluded from the codemap and every sweep — they never enter the map or the
token budget, so no phantom files. Read natively by `codemap.py`.

```
## excluded
edge-trades/           stale untracked clone (produced 369 phantom map files)
storage/framework/**   compiled/cache artifacts
```

## keeps

One row per settled KEEP decision. Audit and the shrink-auditor load these and do
**not** re-flag them (re-verify only if explicitly asked). Shave and audit append
new keeps and hidden-dependency discoveries here — durable, unlike the
per-audit plan that gets superseded each run.

```
## keeps
App\Support\Legacy::render   documented test seam (Feature/RenderTest) — 2026-07
config([...]) keys in Trading  runtime-written; look dead, aren't — 2026-07
NearDupA / NearDupB          deliberate divergent twins (different rounding) — 2026-07
```
